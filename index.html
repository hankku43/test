z-index的限制

最初我的構想是將地鼠洞分為洞內漆黑部分、洞口上方土堆和洞口下方3個部分，並以偽元素(pseudo-elements)的方式組成，加上地鼠後以z-index做出層次，如下左圖所示。然而這樣實際執行起來卻變成右圖的樣子。

探究其原因後發現，我在撰寫地鼠和地鼠洞的HTML程式碼時，編排兩者層狀結構是在同一父元素之下的並列子元素，而作為土堆的偽元素則是地鼠洞子元素。也就是下面所寫的這樣：

 <!-- HTML -->
<div class="set">
    <div class="hole"></div>
    <div class="gopher"><img src="gopher.png" alt="gopher"></div>
</div>
/*CSS*/
.gopher{
    position: absolute;
    z-index: 3;
}
.hole{
    position: absolute;
    z-index: 2;
}
.hole::before{
    content: "";
    position: absolute;
    background-image: url(downhole.png);
    top: 0px;
    z-index: 4;
}
.hole::after{
    content: "";
    position: absolute;
    background-image: url(uphole.png);
    bottom: 0px;
    z-index: 1;}

當程式碼如此撰寫時，兩個偽元素屬於其父元素.hole的子元素，而子元素的z-index並不能覆蓋其父元素的層級設定。說的白話一點，.hole的z-index值為2而.gopher的z-index值為3，因此不論.hole下的子元素其z-index值為多少，就算設為100也不會顯示在.gopher前面，因其父元素.hole的index值就是小於.gopher。

要解決以上問題其實非常容易，有兩個方式可以解決。第一是讓.gopher直接移到.hole之中，讓.gopher也一併成為.hole的子元素；第二是直接刪除.hole的z-index設定，既然父元素沒有z-index值，也就不會綁定其子元素的z-index優先度了。

除了標籤階層問題，還有另一個CSS屬性也會影響z-index的設定，那就是transform。仔細看的話會發現我後3排的地鼠和地鼠洞因應背景的透視有做縮小。但如果使用transform: scale()屬性達成縮小目的的話，會造成以下的狀況發生：

使用transform屬性彷彿會將其子元素全部打包，雖然彼此之間仍可以照z-index設定堆疊，但就無法跟設定transform屬性主體以外的標籤產生正確z-index互動了。而我目前沒有找到直接的解決辦法，而是直接放棄使用transform屬性，直接去修改標籤的長寬達成縮小的效果。

屬標圖案變換

根據我查閱資料，屬鼠標變換有兩種主要的方式，一種就是直接去更改鼠標的圖案，一種是將鼠標圖案隱藏後，新增一個會追隨滑鼠移動的圖示。而我使用的方式是前者，原因是在我嘗試後者之後，發現追蹤滑鼠位置的圖片會有稍微延遲的狀況，向下面所展示的一樣。

而一個打地鼠遊戲最重要的就是反應速度，顯然不能讓鼠標延遲的狀況發生，因此我選擇使用直接更改鼠標的方式。然而直接更改鼠標圖示也有需要注意的地方，那就是圖檔大小。

步驟二

有了單元角鋼，將其複製延長成想要的長度，並旋轉到對應角度，便可以微調參數m檢查各個鎖孔有無對齊，從底下影片中可以清楚看到角鋼交會處的鎖孔皆是相通的。 
module polynumber(num) {        //建立具有num個角鋼單元長度的角鋼條模型
for (i = [0 : num-1])
translate([ 0, uy*i, 0 ]) children();
}
module coner (x,y,H,m) {        //建立角鋼架4角直立柱模型
rotate([90,0,90])polynumber(H){ unit(m);}
translate([x,0,0]) mirror([1,0,0]) rotate([90,0,90])polynumber(H){ unit(m);}
translate([0,y,0]) mirror([0,1,0]) rotate([90,0,90])polynumber(H){ unit(m);}
translate([x,y,0]) rotate([90,0,270])polynumber(H){ unit(m);}
}
module frame(xn,yn,xm,ym){      //建立角鋼橫條模型
translate([T+T,T,ux]) mirror([1,0,0])   rotate([270,90,0]) polynumber(xn) unit(xm);
translate([T+T,uy*yn-T,ux]) rotate([0,90,270]) polynumber(xn) unit(xm);
difference(){
translate([T,T,ux+T])rotate([0,90,0])    polynumber(yn) unit(ym);
translate([T,T+(yn*uy)-T-T,ux+T])rotate([0,90,0])cube([ux,uy,uz]);
}
difference(){
translate([uy*xn+T+T+T,T,ux+T])mirror([1,0,0])rotate([0,90,0])polynumber(yn) unit(ym);
translate([uy*xn+T+T+T,T+(yn*uy)-T-T,ux+T])mirror([1,0,0])rotate([0,90,0])cube([ux,uy,uz]);
}
}
